# FutureForward Wellness: AI-Integrated Stress Relief Prototype
Prototype link: [Live prototype in Codepen][https://codepen.io/editor/Jeyaditya/pen/019d90d8-808a-7fce-bebb-67a429d9ae43?file=%2Findex.html&orientation=top&panel=files&show=preview]
## Project Overview
This project is an interactive, browser-based prototype designed to combat workplace stress and 'Technical Burnout.' It uses simulated AI biometric analysis to dynamically adjust sensory environments (light, sound, aroma) for professional users in high-stress corporate environments.

## Design Thinking Process
This prototype was developed following a rigorous Design Thinking methodology:
1. **Empathize & Define:** Conducted primary research via Google Form surveys to identify core stress archetypes, resulting in detailed User Personas and Empathy Maps (see `/docs` folder).
2. **Ideate:** Utilized digital whiteboarding to map out sensory solutions and workflow integrations, prioritizing high-impact features.
3. **Prototype:** Built a high-fidelity, interactive HTML/CSS/JS frontend application featuring complex state management and customized UI elements.
4. **Test:** Conducted usability testing to gather feedback, resulting in iterative technical improvements (e.g., SVG icon standardization, secure logout routing, and Web Audio API constraints).

## Compulsory Features Included
1. **MoodSync AI:** Dynamic configuration of ambient pod settings based on simulated biometric data.
2. **Breathe with Me:** A synchronized 4-7-8 breathing animation with Web Audio API frequency guidance.
3. **Pod Scheduler:** Calendar integration concept for seamless workday wellness gap detection.
4. **Post-Session Journal:** A reflective input matrix for AI learning and mood tracking.
5. **Stress Tracking Dashboard:** Real-time biometric visualization (HRV/Stress levels).

## Technical Details
* **Frontend:** Built entirely with HTML5, vanilla JavaScript, and Tailwind CSS.
* **Animations:** Features custom CSS keyframe animations for breathing cycles and UI transitions.
* **Audio Engine:** Utilizes the Web Audio API for interactive sound generation, implementing a 'Silent Unlock' method for browser autoplay compliance.
* **UI/UX:** Responsive 'Glassmorphism' design optimized for modern mobile-viewpoint displays.

## How to Run
Simply open `index.html` in any modern web browser (Chrome/Safari recommended). No local server or dependencies are required.
